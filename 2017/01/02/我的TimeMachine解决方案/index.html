我的TimeMachine解决方案

发表于 2017-01-02 |

前言

数据一定是无价的, 尤其是当你需要他时!

我的解决方案

基本情况

我有一台2015mid的macbook pro. 256G除去一些不需要备份的路径(如下图), 大约200G.

所以备份盘大小大约在500G左右就够了.

几种方案

这里直接说几点结论, 或许能够帮到你:

  1. 如果你有Apple专用的硬件, 那就不需要看这篇文章了.
  2. 如果你有群辉, 可以直接使用他的TimeMachine功能.
  3. 如果你有个人PC, 建议使用windows+虚拟机+netatalk方案.

对自建NAS有兴趣的同学可以参考这篇知乎:自己家里搭建NAS服务器有什么好方案？

备份机

  1. 主板+CPU是华擎的J3455-itx(10W, 静音, 性能足够)
  2. 金士顿骇客ddr1600 8G*1(骇客相对体质好一点)
  3. SFX300W铜牌电源(这个sfx电源可选的实在不多)
  4. 机箱是银欣 CS01S(这个是我最满意的, itx机箱, 2*3.5+2*2.5, 关键是非常美观)

网络环境

  1. 电信下行100Mbps, 上行10Mbps
  2. 千兆路由器

注意: 请检查自家的网线是否为超5类. 否则无论多牛逼的设备, 最多只能跑满10MB/s

操作系统

  1. 备份机装了windows10
  2. 虚拟机用了vmware12
  3. 虚拟机系统使用ubuntu16

搭建

安装&配置Netatalk

Netatalk 是一个免费开源的 AppleTalk 通信协议的实现，Linux 或者 BSD 系统通过它可以充当 Mac 的文件服务器 (AppleShare File Server, 网络协议是 AFP)、AppleTalk 路由、打印服务器等。

安装完毕后, 修改配置文件: /etc/netatak/AppleVolumes.default

  1. 注释掉~一行, 否则会把整个用户的目录给共享出去
  2. “TimeMachine”是最终mac读取的根文件夹名称
  3. options:tm告诉mac可以用于timemachine

最后启动服务:

Mac操作

  1. 在Finder中Command+K, 输入afp://你的虚拟机内网IP.

  2. 通过鉴权之后, 你就可以在Finder中发现已经挂载了目录.

  3. 在TimeMachine配置中(添加或移除备份磁盘)找到刚才的目录.

  4. 开始备份.

其他

让虚拟机自动启动

创建一个快捷方式, 属性-目标中添加以下内容, 前半段是你vmware.exe的路径, 后半段是虚拟机的路径, -x会让vmware自动启动主体及虚拟机.

最后把快捷方式, 放到windows的启动目录中就可以了.

关于外网访问

我之前其实一直希望能够在外网能够访问, 我先说一下我的结论

网络太不稳定了

以下需要做的几件事情:

  1. 路由器配置一个ddns, 拥有一个可以找到你家路由器的域名
  2. 将NAS机(虚拟机)的网卡与IP绑定, 防止IP因为DHCP原因改变.
  3. 开启远程桌面, 以防万一还能连上NAS机.
  4. 绑定端口. 以目前万恶的电信及网络安全, 在路由器中绑定比如111->NasIP:3389(远程桌面), 112->Nas虚拟机IP:548(afp)
